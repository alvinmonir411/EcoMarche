# EcoMarche Project Rules

## Project Stack
- Frontend: Next.js 16.2
- Backend: Latest NestJS
- Database: Neon Db PostgreSQL
- Language: TypeScript

## Project Type
EcoMarche is a dress item e-commerce website.

## Coding Rules
- Write easy, clean, beginner-friendly code.
- Do not write complex code.
- Use simple folder structure.
- Use reusable components.
- Use TypeScript properly.
- Keep frontend and backend separate.
- Use clear variable names.
- Add comments only when needed.

## Design Rules
- Modern fashion e-commerce design.
- Mobile responsive layout.
- Clean UI.
- Use simple colors.
- Product images should be highlighted.

## Frontend Rules
- Use Next.js App Router.
- Use Tailwind CSS.
- Create small components.
- Keep pages clean.
- Use server components where simple.
- Use client components only when interaction is needed.

## Backend Rules
- Use NestJS modules.
- Use DTOs.
- Use validation.
- Use type ORM.
- Keep controller, service, and module clean.
- Write simple CRUD APIs first.

## Database Rules
- Use PostgreSQL.
- Products must support size, color, price, stock, and images.
- Orders must include customer info, items, payment status, and delivery status.

## First Goal
Build basic e-commerce flow:
Home → Shop → Product Details → Cart → Checkout → Order
